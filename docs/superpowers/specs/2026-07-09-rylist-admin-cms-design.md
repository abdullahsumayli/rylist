## ١) المشكلة والهدف

الموقع الحالي ([index.html](../../../index.html) وأخواته) موقع **ثابت** ممتاز في السرعة والأمان والـSEO، لكنه:
- **ساكن:** المحتوى مكتوب يدويًا في [assets/js/data.js](../../../assets/js/data.js) — تعديله يحتاج مبرمجًا.
- **أعمى:** نموذج «سجّل اهتمامك» يفتح واتساب/بريد ([main.js:275-285](../../../assets/js/main.js#L275-L285)) و**يضيع الليد** — لا يُحفظ ولا يُقاس.
- **مخفيّ جزئيًا:** تبديل اللغة client-side ([i18n.js](../../../assets/js/i18n.js)) بلا `/en/` حقيقية ولا hreflang → جوجل يفهرس العربي فقط.

**الهدف:** إضافة لوحة تحكم كاملة يديرها غير المبرمج (٣ أدمن) لكل محتوى الموقع، مع التقاط العملاء وقياسهم، وSEO متعدّد اللغات حقيقي — **مع الاحتفاظ بموقع عام ثابت وسريع وآمن** (لا نتحوّل لووردبريس).

## ٣) المعمارية (المرحلة ١)

النظام جزآن منفصلان يتشاركان قاعدة واحدة:

```
  ┌─────────────────────────┐        ┌──────────────────────────────┐
  │  لوحة التحكم /admin/     │  RW    │                              │
  │  (JS صِرف + supabase-js) │ ─────▶ │        Supabase              │
  │  دخول · CRUD · صور · نشر │        │  Postgres + Auth + Storage   │
  └─────────────────────────┘        │  + Edge Functions            │
                                      └──────────────┬───────────────┘
   زر «نشر» ▶ Edge Function `publish` ──▶ Vercel Deploy Hook
                                                     │ build (service_role key يقرأ الكل)
                                                     ▼
  ┌──────────────────────────────────────────────────────────────┐
  │  الموقع العام الثابت (Vercel) — سريع · SEO · آمن               │
  │  / (عربي)   /en/ (إنجليزي)   /zh/ (صيني، عند التفعيل)          │
  │  data.js مُعاد توليده · صفحات لكل عقار · hreflang · sitemap    │
  │  نموذج الطلبات + أيقونة واتساب عائمة  ──INSERT──▶ leads        │
  └──────────────────────────────────────────────────────────────┘
```

- **الخلفية:** Supabase (Postgres + Auth + Storage + Edge Functions).
- **الاستضافة:** Vercel (build hook للنشر).
- **لوحة التحكم:** صفحات ثابتة تحت `/admin/`، JavaScript صِرف + `@supabase/supabase-js` عبر ESM CDN — **بلا build**، متسقة مع ستاك الموقع.
- **الموقع العام:** يبقى ثابتًا. لا يقرأ من القاعدة وقت التشغيل؛ محتواه يُولَّد وقت **النشر**. الاستثناء الوحيد: نموذج الطلبات يكتب مباشرة في `leads` (باستخدام المفتاح المنشور + سياسة RLS).

## ٤) نموذج البيانات (Supabase / Postgres)

**مبدأ متعدّد المحليّات (locale-driven):** الحقول القابلة للترجمة تُخزَّن كـ`jsonb` مفاتيحه رموز اللغات — إضافة لغة = إضافة مفتاح، **بلا تغيير مخطّط**. الحقول الآلية (المفاتيح، الأسعار، الحالات) أعمدة عادية.

### الجداول

**`locales`** — اللغات المتاحة
| عمود | نوع | ملاحظات |
|---|---|---|
| `code` | text PK | `ar` · `en` · `zh` |
| `name` | text | الاسم المعروض للّغة |
| `dir` | text | `rtl` \| `ltr` |
| `enabled` | boolean | الصيني `false` حتى يجهز محتواه |
| `sort_order` | int | ترتيب المبدّل |

|---|---|---|

| `sort_order` | int | |

**`projects`** — العقارات/المشاريع

## ٥) الأمان (Auth + RLS)

- **Auth:** Supabase Auth بالبريد + كلمة المرور. **التسجيل العام معطّل.** يُنشأ الـ٣ أدمن يدويًا (Console أو دعوة)، ويُضاف كل واحد لجدول `admins`.
- **دالة مساعدة:** `is_admin()` = هل `auth.uid()` موجود في `admins`.
- **RLS مفعّل على كل الجداول:**

| الجدول | anon (زائر) | authenticated + is_admin |
|---|---|---|
| `leads` | **INSERT فقط** | SELECT / UPDATE / DELETE |
| كل جداول المحتوى (`projects`, `news`, `partners`, `stats`, `contact`, `pages`, `social_links`, `taxonomies`, `locales`) | **لا شيء** | SELECT / INSERT / UPDATE / DELETE |
| `admins` | لا شيء | SELECT (والتعديل عبر service_role فقط) |

- **لماذا anon محروم من قراءة المحتوى؟** لأن الموقع العام **ثابت** ويقرأ من `data.js` المُولَّد، لا من القاعدة وقت التشغيل. القارئ الوحيد للمحتوى هو **سكربت البناء** بمفتاح `service_role` (يتخطّى RLS، ويعيش في بيئة Vercel فقط). هذا أضيق سطح ممكن.
- **المفاتيح:**
  - المفتاح **المنشور (publishable/anon)** يُستخدم في متصفّح لوحة التحكم والموقع العام — آمن، لأن RLS يحمي البيانات (بعد الدخول يُطبَّق JWT المستخدم فتنطبق سياسات الأدمن).
  - المفتاح **service_role** = سرّ Vercel، للبناء فقط. لا يظهر في أي متصفّح.
  - `ANTHROPIC_API_KEY` (للمراحل ٢–٤) = سرّ Supabase، لا ينكشف أبدًا.

## ٦) التخزين (Storage)

- Bucket باسم `media`: **قراءة عامة** (الصور تُشار إليها بروابط عامة من الموقع الثابت)، **كتابة للمصادَق عليهم فقط** (رفع الأدمن).
- الأدمن يرفع الصور من نماذج CRUD؛ يُحفظ `image_url` في السجل.

## ٧) لوحة التحكم (`/admin/`)

- صفحات ثابتة تحت `/admin/`، JS صِرف + `@supabase/supabase-js` (ESM CDN)، تعيد استخدام توكِنز [styles.css](../../../assets/css/styles.css) + `admin.css`.
- **صفحة الدخول:** بريد + كلمة مرور (Supabase Auth). لا استرجاع/تسجيل عام.
- **الهيكل بعد الدخول:** قائمة جانبية + منطقة محتوى.
- **الأقسام:**
  1. **لوحة رئيسية:** صندوق الطلبات (جدول `leads` مع تغيير الحالة جديد/تم التواصل/مغلق) + أرقام سريعة (عدد المشاريع، الطلبات الجديدة…).
  2. **العقارات · الأخبار · الشركاء · الأرقام · الصفحات:** جدول قائمة + نموذج إضافة/تعديل/حذف. الحقول القابلة للترجمة تُعرض بتبويبات لغة (`ع / EN / 中文`) لكل locale مفعّل.
  3. **التواصل · السوشيال ميديا:** تحرير مباشر.
  4. **اللغات:** تفعيل/تعطيل locale (زر يقلب `locales.enabled` — به نُطلق الصيني عند الجهوز).
  5. **زر «نشر»:** يستدعي Edge Function `publish`.
- **رفع الصور:** widget يرفع لـStorage ويعبّئ `image_url`.
- **الأمان:** كل عمليات اللوحة تمرّ عبر RLS (أدمن فقط). واجهة اللوحة لا تحمل أي مفتاح سرّي.

## ٨) الموقع العام — التغييرات

1. **متعدّد المحليّات حقيقي (SEO):** بدل التبديل client-side، **نُولّد صفحات لكل locale مفعّل** عند البناء:
   - `/` (عربي، الافتراضي) · `/en/…` · `/zh/…` (عند التفعيل).
   - كل صفحة محتواها مخبوز باللغة الصحيحة + وسوم `hreflang` + `canonical`.
   - المبدّل يصير **روابط حقيقية** بين نسخ اللغة (`ع / EN / 中文`)، لا مجرّد JS.
2. **صفحة لكل عقار:** `/projects/<code>.html` (وبكل locale) — مدخل SEO + وجهة إعلان.
3. **أيقونة واتساب عائمة:** زر ثابت بالزاوية في كل الصفحات → `wa.me/<contact.whatsapp>`.
4. **روابط السوشيال ميديا:** تُعرض في الفوتر (والشريط عند اللزوم) من `social_links` المُولَّدة.
5. **نموذج الطلبات:** يكتب مباشرة في `leads` (INSERT عبر anon) بدل فتح واتساب فقط — مع إبقاء زر واتساب كخيار. يُخزَّن `source='form'` و`locale`.

> ملاحظة تنفيذ: الصفحات الست الحالية تتحوّل إلى **قوالب** يملؤها البناء لكل locale. المحتوى الديناميكي (عقارات/أخبار/شركاء/أرقام) من `data.js` المُولَّد؛ النصوص الطويلة (من نحن/الخدمات) من جدول `pages`.

## ٩) النشر → التوليد الثابت

- **سكربت البناء** `scripts/build.mjs` (Node، يعتمد على `@supabase/supabase-js` + مكتبة Node القياسية فقط):
  1. يقرأ كل المحتوى من Supabase بمفتاح `service_role` (من متغيّر بيئة Vercel).
  2. يقرأ `locales` المفعّلة، ويولّد **فقط** للّغات `enabled=true`.
  3. يُنتج: `assets/js/data.js` المُعاد توليده · صفحات HTML لكل locale · صفحة لكل عقار لكل locale · `sitemap.xml` مع hreflang · `robots.txt`.
- **زر «نشر»** في اللوحة → Edge Function `publish` → POST إلى **Vercel Deploy Hook** (رابطه سرّ Supabase) → Vercel يشغّل `build.mjs` وينشر.
- **إطلاق الصيني لاحقًا:** قلب `locales.enabled` للصيني إلى `true` ثم «نشر» → البناء يولّد `/zh/`. بلا إعادة بناء للنظام.
- الطلبات (`leads`) تُكتب مباشرة، فصندوق الوارد فوري بلا انتظار نشر.

## ١٠) معايير القبول (المرحلة ١)

- **AC-1.1** ٣ أدمن يقدرون الدخول؛ لا تسجيل عام؛ غير المصادَق عليه لا يقرأ أي محتوى ولا الطلبات.
- **AC-1.2** الأدمن يضيف/يعدّل/يحذف: العقارات، الأخبار، الشركاء، الأرقام، التواصل، الصفحات، السوشيال — كلها من اللوحة، بلا كود.
- **AC-1.3** الحقول القابلة للترجمة تُحرَّر لكل locale مفعّل عبر تبويبات لغة، ومخزّنة كـ`jsonb` (إضافة لغة لا تغيّر المخطّط).
- **AC-1.4** رفع صورة للعقار/المقال يعمل ويُخزَّن في Storage ويُشار إليه بـ`image_url`.
- **AC-1.5** نموذج «سجّل اهتمامك» يحفظ صفًّا في `leads` (`source='form'`, `locale`)، ويظهر فورًا في صندوق الطلبات مع تغيير الحالة.
- **AC-1.6** زر «نشر» يشغّل بناء Vercel، وخلال ~دقيقة يظهر التغيير على الموقع الثابت.
- **AC-1.7** الموقع يُولّد `/` و`/en/` (والصيني عند التفعيل) بصفحات ثابتة حقيقية + `hreflang` + `canonical`، وصفحة لكل عقار.
- **AC-1.8** أيقونة واتساب عائمة + روابط السوشيال تظهر في كل الصفحات وتعمل.
- **AC-1.9** تفعيل/تعطيل الصيني من اللوحة يظهر/يخفي `/zh/` عند النشر التالي.
- **AC-1.10** الموقع العام يبقى ثابتًا (لا قراءة من القاعدة وقت التشغيل عدا INSERT الطلبات)، ويعمل بلا أي مفتاح سرّي في المتصفّح.
