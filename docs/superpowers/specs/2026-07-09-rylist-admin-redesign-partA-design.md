## ٠) السياق ضمن الرؤية الكبرى

هذا **الجزء أ** من ٣ أجزاء متتابعة لترقية اللوحة (كل جزء يُنشر لحاله):

| الجزء | المحتوى | الحالة |
|---|---|---|
| **أ — الشكل الجديد + حالة «محجوز»** | إعادة تصميم اللوحة بالكامل بهوية RYLIST + قسم العقارات الغني (بطاقات إحصائية/شبكة/فلاتر) + إضافة حالة `reserved` | **هذا الـspec** |
| ب — المساعد الذكي ✨ | المرحلة ٢: أزرار ✨ (توليد/تحسين/ترجمة/عنوان‑SEO/أفكار) عبر Claude Opus 4.8 | spec لاحق |
| ج — الاستيراد والبروشور | رفع دفعة من Excel/CSV + بروشور PDF لكل عقار | spec لاحق |

**مبدأ حاكم:** الجزء أ يبني *الهيكل والنماذج* التي تسكن فيها ميزات ب وج، فلا نُعيد الشغل. النماذج تترك **نقاط تركيب** (mount points) فارغة لأزرار ✨ (ب) وأزرار الاستيراد/البروشور (ج) تُملأ لاحقًا.

## ١) المشكلة والهدف

**المشكلة:** لوحة المرحلة ١ ([admin/](../../../admin/)) وظيفية لكن بدائية بصريًا — جداول عارية، بلا نظرة سريعة على الحالة، وتجربة إضافة عقار جافة. المستخدم يريد لوحة أنيقة عملية على مستوى منتجات SaaS الحديثة (استلهم من «صامل الذكي»).

**الهدف:** إعادة تصميم اللوحة بالكامل بهوية RYLIST («ترف صامت»)، مع قسم عقارات غني (بطاقات إحصائية + شبكة بطاقات بصور + بحث/فلاتر + نموذج أنيق)، وإضافة حالة **«محجوز»** — **دون كسر** الموقع الثابت أو تدفّق النشر أو RLS، وبنفس قيد «بلا build» للّوحة.

## ٢) نظام التصميم (من الموكب-أب المعتمد)

مبني على توكِنز RYLIST الحالية في [assets/css/styles.css](../../../assets/css/styles.css) — **لا نخترع هوية جديدة**:

- **الألوان:** حِبر `#17140F` (القائمة الجانبية الداكنة) · عاجي `#F6F3EB` (الخلفية) · أبيض دافئ `#FBF9F4` (البطاقات) · عاجي ناعم `#EFEBE0` (الحقول) · شامبين `#A38A58` / ذهبي عميق `#8E763F` (اللمسة والأزرار الأساسية).
- **دلالات الحالة (منفصلة عن لون اللمسة):** متاح = أخضر `#4E6A4E` · محجوز = كهرماني `#A16C2E` · مباع = طيني `#8A4B3D`.
- **الخط:** `IBM Plex Sans Arabic` (مع بدائل نظام) لكامل الواجهة؛ ووردمارك `RYLIST` بخط `Georgia` serif.
- **الوضعان:** فاتح وداكن عبر توكِنز CSS (`prefers-color-scheme` + `data-theme`).
- **التخطيط:** قائمة جانبية داكنة ثابتة (يمين، RTL) + منطقة عمل قابلة للتمرير. زوايا ناعمة معتدلة (`--radius` ~8–14px، أنعم قليلًا من زوايا الموقع العام الحادّة لأنها أداة لا صفحة تحريرية).

## ٣) المعمارية (بنية ملفات `/admin/`)

يبقى القيد: **صفحات ثابتة، JavaScript صِرف (ESM)، بلا build**. نُعيد تنظيم الوحدات لتكون كل وحدة لها غرض واحد واضح:

```
admin/
  index.html      — هيكل القشرة (login + shell: sidebar + main)
  config.js        — (كما هو) URL/anon key/locales
  db.js            — عميل supabase + دوال auth  [انظر §٧ التحسين]
  shell.js         — [جديد] القائمة الجانبية + التوجيه (router) + رأس الصفحة
  entities.js      — (كما هو تقريبًا) تعريف الكيانات + إضافة 'reserved'
  fields.js        — [جديد] نموذج مشترك: حقول + تبويبات لغة + نقاط تركيب ✨
  projects.js      — [جديد] قسم العقارات الغني: بطاقات إحصائية + فلاتر + شبكة بطاقات
  list.js          — [جديد، مستخرج من ui.js] قائمة جدولية عامة لبقية الكيانات (بالستايل الجديد)
  leads.js         — (كما هو، بإعادة تنسيق) صندوق الطلبات
  publish.js       — (كما هو) زر النشر
  admin.css        — [إعادة كتابة كاملة] نظام التصميم أعلاه
```

- **`shell.js`**: يرسم القائمة الجانبية من قائمة أقسام معرّفة، يدير التوجيه (يستبدل محتوى `<main>` بلا إعادة تحميل)، يرسم الرأس (breadcrumb) وزر النشر والخروج.
- **`projects.js`**: يجلب `projects`، يحسب البطاقات الإحصائية (إجمالي/متاح/محجوز/مباع)، يرسم شريط البحث + شرائح الفلاتر، ويعرض شبكة بطاقات (صورة/شارة/سعر/نسبة بيع/مميّز). الفلترة والبحث client-side على البيانات المجلوبة. أزرار البطاقة: تعديل · ✨ (نقطة تركيب لـب) · حذف.
- **`fields.js`**: مكوّن نموذج مشترك يستهلكه `projects.js` و`list.js` — يبني الحقول من تعريف الكيان، تبويبات اللغة (`ع/EN/中文`) للحقول i18n، ورفع الصور. يحوي **حاوية شريط ✨** فارغة (تُملأ في الجزء ب).
- **`list.js`**: عرض جدولي عام لبقية الكيانات (أخبار/شركاء/أرقام/صفحات/تواصل/سوشيال) بالستايل الجديد، يعيد استخدام `fields.js` للنماذج.

**الحدود واضحة:** كل وحدة يمكن فهمها واختبارها وحدها؛ `shell` لا يعرف تفاصيل الأقسام، `projects`/`list` لا يعرفان التوجيه، `fields` لا يعرف أي كيان بعينه.

## ٤) تغيير نموذج البيانات — حالة «محجوز»

العمود `projects.status` حاليًا: `check (status in ('available','sold'))` مع افتراضي `'available'` ([phase1a](../../plans/2026-07-09-rylist-phase1a-supabase-backend.md#L171)).

**هجرة `0006_reserved_status`:** إسقاط قيد التحقّق الحالي وإضافة قيد يسمح بالقيم الثلاث `('available','reserved','sold')`. لا تغيير على الصفوف الموجودة (تبقى `available`/`sold`).

**التأثيرات التبعية (ضمن نطاق الجزء أ):**
1. **اللوحة:** `entities.js` — خيارات الحالة تصير `['available','reserved','sold']` بتسميات عربية.
2. **الموقع العام:** تصيير شارة «محجوز» في بطاقة العقار وصفحة العقار (كهرماني)، وترجمتها في `data.js`/القوالب. العقار المحجوز **يبقى ظاهرًا** للزوّار بشارته.
3. **البطاقات الإحصائية:** تحسب المحجوز.

## ٥) الحفاظ على ما يعمل (لا انحدار)

- **RLS/الأمان:** بلا تغيير — كل القراءة/الكتابة عبر عميل anon + جلسة الأدمن (سياسات `is_admin()`). لا مفتاح سرّي في المتصفّح.
- **تدفّق النشر:** بلا تغيير — زر «نشر» → Edge Function `publish` (المُصلَح CORS مؤخّرًا) → Vercel.
- **الموقع العام والبناء:** بلا تغيير عدا تصيير شارة «محجوز».
- **كل كيانات المحتوى** تبقى قابلة للإضافة/التعديل/الحذف كما في AC-1.2.

## ٦) معايير القبول (الجزء أ)

- **AC-A.1** قشرة جديدة: قائمة جانبية RTL بالأقسام الفعلية + زر نشر + خروج + هوية المستخدم؛ التنقّل يستبدل المحتوى **بلا إعادة تحميل**؛ تطابق الموكب-أب المعتمد.
- **AC-A.2** قسم العقارات يعرض ٤ بطاقات إحصائية (إجمالي/متاح/محجوز/مباع) محسوبة حيًّا من البيانات.
- **AC-A.3** العقارات تُعرض كبطاقات بصور + شارة حالة + سعر + نسبة بيع + نجمة «مميّز»؛ البحث وشرائح الفلاتر (الكل/متاح/محجوز/مباع) تعمل client-side.
- **AC-A.4** نموذج إضافة/تعديل العقار المعاد تصميمه: تبويبات لغة للحقول i18n، ورفع صورة يعمل، وحفظ/تعديل/حذف عبر RLS.
- **AC-A.5** حالة `reserved` مدعومة طرفًا لطرف: القيد يسمح بها · الأدمن يعيّنها · الموقع/البناء يصيّر شارة «محجوز» · البطاقة الإحصائية تعدّها.
- **AC-A.6** بقية الأقسام (طلبات/أخبار/شركاء/أرقام/صفحات/تواصل/سوشيال) تُعرض بالستايل الجديد وتبقى وظيفية بالكامل.
- **AC-A.7** يعمل بالوضعين الفاتح/الداكن، RTL سليم، وتدفّق النشر بلا تغيير.
- **AC-A.8** بلا انحدار: المحتوى الحالي يظهر، والبناء ينجح، والطلبات تُحفظ وتظهر.

## ٧) تحسين مضمّن موصى به (يعالج هشاشة سابقة)

اللوحة تستورد `supabase-js` وقت التشغيل من `esm.sh` ([db.js](../../../admin/db.js)) — نقطة فشل مفردة (لو تعثّر CDN تتعطّل اللوحة كليًا، وهو المشتبه في «لا تفتح بالوضع الخفي»). **ضمن إعادة الكتابة، نُثبّت نسخة محلية من `@supabase/supabase-js` (vendored) داخل `admin/vendor/` ونستورد منها** بدل CDN — يزيل الاعتماد الخارجي دون إدخال build.

## ٨) خارج نطاق الجزء أ

- المساعد الذكي ✨ (الجزء ب) — لكن `fields.js` يترك حاوية شريط ✨ جاهزة.
